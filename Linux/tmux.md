## tmux

<font color="#ffff00">tmux</font> 는 Terminal Multiplexer 의 약자로, **하나의 터미널 창에서 여러 개의 세션을 만들고 관리할 수 있게 해주는 도구**입니다. 

## 왜 사용하는가

1. 세션 유지 : 네트워크 연결이 끊기거나 터미널 창을 닫아도 **실행 중인 프로세스가 죽지않고 백그라운드에서 계속 돌아갑니다.** (`nohup` 의 상위 호환)
2. 화면 분할 : 하나의 창을 여러 개로 쪼개어 모니터링과 코딩을 동시에 할 수 있습니다.
3. 회사에서 돌려놓은 세션을 집에 가서 그대로 다시 열어 작업할 수 있습니다.

## Installation

```bash
sudo apt update
sudo apt install tmux
```

설치가 완료되었다면 터미널에 아래 명령어를 입력해 버전을 확인해 보세요.

```bash
tmux -V
```
## 세션 내부 단축키

`tmux` 안에서는 모든 명령 앞에 **ctrl + b** 를 먼저 눌러야 합니다.

- d : 현재 세션에서 빠져나오기
- \[ : 마우스 휠 대신 키보드 방향키로 이전 로그를 보고 싶을 때 (빠져나올 땐 q)
- % , " : 화면 세로/가로 분할
- o : 분할 된 화면 간 이동

## 세션 관리 키워드

|**기능**|**명령어**|**비고**|
|---|---|---|
|**새 세션 생성**|`tmux new -s [세션이름]`|이름 없이 만들면 숫자로 자동 지정됨|
|**세션 목록 확인**|`tmux ls`|현재 돌아가고 있는 세션 리스트 출력|
|**세션 다시 접속**|`tmux attach -t [세션이름]`|`a`만 써도 무방 (`tmux a -t ...`)|
|**세션 종료**|`exit` 또는 `Ctrl + d`|세션 내부에서 입력 시 세션 자체가 삭제됨|
|**세션 외부에서 죽이기**|`tmux kill-session -t [세션이름]`|멈춘 세션을 강제 종료할 때 사용|
